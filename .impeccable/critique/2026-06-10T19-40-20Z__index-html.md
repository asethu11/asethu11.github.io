## Overall Impression
A confident, legible, genuinely hand-made portfolio with strong aesthetic restraint. The biggest opportunities are not visual polish but (1) the conversion action — the site's stated goal is recruiters reaching out, yet the contact affordance is the least prominent element; and (2) accessibility robustness — the red accent fails WCAG AA in light mode, and the homepage has no focus styles or reduced-motion handling.

## Priority Issues

- **[P1] Red accent fails WCAG AA in light mode (theme-parity / ui-eval BLOCKING)**: `.section-t` headings #db4f4f = 3.90:1, rotating loader words #fc4a4a = 3.29:1, `.uh-bee` = 3.90:1 — all below 4.5:1 on #FCFCFC. They PASS in dark (5.12 / 6.07). Violates the site's own PRODUCT.md AA commitment. Fix: darken the light-mode `--red` token to ≥4.5:1 (≈ #b83a3a or OKLCH equivalent), or reserve red for genuinely large/bold text only. Command: $impeccable colorize.
- **[P1] The conversion action is hidden**: goal is "reach out (LinkedIn, email)", but email is a tiny envelope icon that silently copies via `href="#"` with no visible label and no `mailto:` fallback. ui-eval Modernity Gate #2 ("one primary action dominates") fails. Fix: add a clearly labelled primary contact affordance; progressive-enhance copy on top of a real `mailto:`. Command: $impeccable clarify.
- **[P2] No keyboard focus states; previews are mouse-only**: zero `:focus`/`:focus-visible` rules in any CSS the homepage loads; `.project-row` previews trigger on `mouseenter` only (shared.js), never on focus. Keyboard / screen-reader users get no feedback and never see the preview panel. Fix: add focus-visible styles mirroring hover; show preview on focus. Command: $impeccable harden.
- **[P2] No reduced-motion handling on homepage**: always-on blinking cursor, auto-typing, infinite word-spin, infinite skeleton shimmer; motion.css is not linked and no `prefers-reduced-motion` guard exists. Fix: add a reduced-motion block that pauses/freezes these. Command: $impeccable animate.
- **[P2] Mobile drops the richest content and apologizes**: "Other projects" previews are `display:none` < 768px, and a "best viewed on desktop" notice tells phone users (many recruiters) the site is suboptimal. Fix: inline thumbnails per row on mobile; drop the apology. Command: $impeccable adapt.
- **[P3] Weak section/title hierarchy**: h2 section labels 17.6px vs h3 project titles 16px = 1.1× (flat); impeccable wants ≥1.25× between steps. Command: $impeccable typeset.
- **[P3] Markup validity in intro**: stray `</p>`, loose text node, `<br><br>` spacing hacks around the About link. Command: $impeccable polish.
- **[P3] Generic alt text**: two thumbnails use alt="Project GIF" (not GIFs, not descriptive). Command: $impeccable harden.

## Questions to Consider

- Should the red accent meet AA in light mode, or is red reserved for decoration only (and section labels move to ink)?
- On mobile, is the text-only "Other projects" list intentional, or should thumbnails ride along?
