On restraint: why one accent is enough.

Most systems collect color the way attics collect things. Ours does the opposite — one blue, five places, and a near-pure-black canvas that lets it land.

P1
Point One Zero 6 min read v0.4.0 reference May 26, 2026

The first version of this system had seven brand variants. A blue for primary, a green for callouts, a coral for warnings, a violet for "new", an amber for trust, and two neutrals that wanted to feel like accents. Every section had its color. Every color had a meaning we had written down once and then forgotten.

The problem was not the colors. The problem was that none of them meant anything anymore.

The temptation of more

There is always a case for a fourth color. A pricing page wants a "best value" tier; that wants a color. An empty state wants warmth; that wants a color. A new-feature flag wants attention; that wants a color. Each request is reasonable in isolation, and yet —

"

Restraint is the only frame that scales. Permission is not.

— a rule we keep on a sticky note

— each request, granted, costs the system its handle on what blue means. The first accent is a signal; the seventh is just chrome. Once color stops carrying meaning, it starts demanding effects — gradients, glows, glass — to do the work color was supposed to do.

What a single accent earns you

When the brand is one color, the color can mean something specific. Ours does: attention. The blue appears where the user should look first, and it appears nowhere else. That predictability is the entire payoff.

Five rules govern where the blue can land:

  1. The brand mark. The dot in the wordmark, and a single brand-dot inside a hero status pill.
  2. One word per hero. A single inline .brand-word in the page's main headline. Used twice across the page, max.
  3. Focus rings. 2px outline at 2px offset. The functional case — accessibility never negotiates.
  4. Link underlines on hover. Text stays the foreground color; the underline is brand. The hover is the signal.
  5. One featured number. The brand-colored numeric in the one featured metric card per page. Never two.

Notice what is not on the list. Buttons are not brand-filled. Eyebrow chips are not blue. Section dividers are not blue. The blue is a noun, not a decoration.

What this looks like in code

The system exposes the rule via a single class. To color the one inline word in a hero, use .brand-word. To put a brand dot before a status pill's text, use .brand-dot. Both are intentionally specific names — .text-brand would have leaked into a hundred places we did not authorize.

/* Use exactly twice per page, max. */
.brand-word { color: var(--brand); }

/* The mark in miniature. Glow is the only allowed glow. */
.brand-dot {
  width: 6px; height: 6px;
  border-radius: 9999px;
  background: var(--brand);
  box-shadow: var(--brand-glow);
}

There is no .bg-brand utility. There is no brand button variant. There is no "primary accent" filter. The shape of the API enforces the shape of the rule.

Naming as enforcement

Tokens are mirrored to the shadcn convention — --background, --foreground, --primary — but the brand color lives in its own slot called --brand, not --accent. That single naming choice keeps the door closed: hover surfaces want to be called accent, and they should be neutral.

Outcomes

Press ⌘K on any of our product surfaces and you will see the system's strictest moment — a popover with a single blue focus ring, on a near-black canvas, with no other color present. That moment is the brand.

By the numbers. The marketing site has roughly 12,000 pixels of brand blue across its entire homepage — every dot, every focus ring, every link underline, every one of the four featured-metric digits. The dark canvas takes the rest.

We measured what restraint earned us. The number is small, but the point is the ratio.

If the design language has any one rule, it is this: every accent must justify its place by carrying a meaning the rest of the canvas cannot. Color is the smallest unit of communication. Spend it once, and spend it where it counts.
